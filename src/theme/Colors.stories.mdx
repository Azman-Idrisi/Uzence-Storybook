## Usage Guidelines

### Accessibility

- All color combinations meet WCAG 2.1 AA standards for contrast
- Text colors are carefully chosen for readability
- Interactive elements have distinct hover and focus states
- Semantic colors are used consistently for user feedback

### Best Practices

1. Use semantic colors for their intended purpose:
   - Success: Positive actions, completion states
   - Warning: Cautionary actions, important notices
   - Error: Critical issues, destructive actions
   - Info: Neutral information, help text

2. Maintain consistent use of surface layers:
   - Primary: Main component backgrounds
   - Secondary: Nested components, cards
   - Tertiary: Elevated components, modals

3. Text hierarchy:
   - Primary: Main content, headings
   - Secondary: Supporting text
   - Tertiary: Meta information
   - Disabled: Inactive elements

4. Brand colors:
   - Use primary brand color for main CTAs
   - Secondary brand color for complementary actions
   - Tertiary brand color for accents and illustrations

### CSS Variables

All colors are available as CSS variables:

```css
/* Surface & Background */
var(--surface-primary)
var(--surface-secondary)
var(--surface-tertiary)

/* Text Colors */
var(--text-primary)
var(--text-secondary)
var(--text-tertiary)
var(--text-disabled)

/* Brand Colors */
var(--brand-primary)
var(--brand-primary-hover)
var(--brand-primary-active)
var(--brand-primary-subtle)

/* Semantic Colors */
var(--semantic-success)
var(--semantic-warning)
var(--semantic-error)
var(--semantic-info)
```
