# Tag Component

The Tag component is used to categorize content, display statuses, or highlight information in a compact format.

## Anatomy

The Tag component consists of:
- An optional leading icon
- Label text
- An optional trailing icon
- Visual states (default, hover, disabled)

## Usage

Tags are useful for:
- Categorizing content
- Indicating status
- Highlighting metadata
- Filtering content

```tsx
import { Tag } from './Tag';
import { Check } from 'lucide-react';

// Basic usage
<Tag>Default</Tag>

// With variant
<Tag variant="success">Success</Tag>

// With icon
<Tag icon={Check}>Completed</Tag>

// Disabled state
<Tag disabled>Unavailable</Tag>
```

## Accessibility

- Uses appropriate ARIA roles (`status`)
- Includes `aria-disabled` when in disabled state
- Has proper focus states for keyboard users
- Ensures color contrast meets WCAG AA standards
- Icon elements include appropriate sizing for readability

## Props

| Prop | Type | Default | Description |
| ---- | ---- | ------- | ----------- |
| variant | 'primary' \| 'success' \| 'error' \| 'warning' | 'primary' | The visual style of the tag |
| size | 'small' \| 'medium' \| 'large' | 'medium' | The size of the tag |
| disabled | boolean | false | Whether the tag is disabled |
| icon | LucideIcon | undefined | Optional icon to display before text |
| trailingIcon | LucideIcon | undefined | Optional icon to display after text |
| className | string | undefined | Additional CSS classes to apply |

Plus all standard HTML attributes for a `span` element.
